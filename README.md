# 广附学情数据看板

广州大学附属中学教育集团 **学情数据看板** 的静态站点：登录、驾驶舱、诊断分析、学生管理与学生详情等页面，数据来自 `data/` 目录下的 CSV（演示或正式数据需自行替换）。

## 功能概览

| 页面 | 文件 | 说明 |
|------|------|------|
| 登录 | `index.html` | 统一登录入口 |
| 驾驶舱 | `cockpit.html` | 总览指标与图表 |
| 班级管理 | `diagnosis.html` 或 `class-board.html` | 核心指标 + 作业分析 + 自主学习（两页内容相同） |
| 学生管理 | `students.html` | 学生列表 |
| 学生详情 | `student-detail.html` | 个人学情与资源使用 |

前端为 **HTML / CSS / JavaScript**，图表与 CSV 解析依赖页面中通过 **jsDelivr** 引入的 **ECharts**、**PapaParse**（部署环境需能访问公网 CDN，或自行改为内网资源地址）。

## 本地运行

1. **不要**用资源管理器直接双击打开 HTML（`file://` 协议下浏览器会限制读取本地 CSV，登录与图表可能异常）。
2. 使用本地 HTTP 服务，任选其一：
   - 双击 `start-local.bat`（Windows）
   - 或执行 `start-local.ps1` / `serve-local.py`（见仓库内脚本说明）
3. 在浏览器中访问控制台输出的地址（保持服务进程不关闭）。

更完整的验收步骤与「与线上一致」的约定见仓库内 **`部署与验收说明.txt`**。

## 子路径部署（可选）

若站点挂在子路径（例如 `https://example.com/edu/board/`），请编辑 **`js/site-config.js`**，将 `GUANGDA_SITE_BASE` 设为与线上一致的路径（如 `'/edu/board/'`）。部署在域名根路径时保持 `''` 即可。

## 部署到 Vercel

1. 将包含本看板的仓库推送到 GitHub。
2. 在 [Vercel](https://vercel.com) 中 Import 该仓库。
3. **Root Directory**：
   - 若 **仓库根目录就是本目录**（根下直接有 `index.html`、`css`、`js` 等），留空或填 **`./`**；
   - 若看板在仓库里的**子文件夹**中，则填该子文件夹名称（本地开发请使用 **`guangfu`** 目录下的 `start-local.bat`）。
4. 无需填写 Build Command；静态资源由平台直接托管。
5. 推送新提交后会自动触发重新部署。

## 数据与安全

- 业务数据位于 **`data/*.csv`**，可按需替换为真实或脱敏数据。
- 若 CSV 中含敏感信息（如账号表），请使用 **私有仓库**，并避免将密钥、内部账号写入仓库或提交记录。

## 维护目录说明

**请只在 `guangfu` 目录内修改与启动看板**（`start-local.bat` / `serve-local.py`）。同级其他文件夹不是当前交付版本，勿从那里启动或验收。
